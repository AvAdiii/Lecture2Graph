# Lecture2Graph

**Neuro-symbolic extraction of concept-prerequisite DAGs from code-mixed lecture videos — with a reproducible evaluation benchmark.**

Lecture2Graph watches a (Hindi/English code-mixed) lecture video and produces a
**directed acyclic graph of the concepts it teaches and the order they depend on
each other** — a study roadmap, recovered automatically from speech + the
handwritten board.

It does this three ways and **measures which wins**:

| method | what it is | edge P | edge R | **edge F1** | node F1 | order acc |
|---|---|---|---|---|---|---|
| symbolic | regex + hand-authored domain rules + DAG inference | **0.549** | 0.619 | 0.540 | **0.833** | 0.921 |
| neural | local LLM (qwen2.5-7B) reads the transcript | 0.289 | 0.135 | 0.174 | 0.479 | 0.527 |
| **hybrid** | **fuse the two (this repo's contribution)** | 0.485 | **0.664** | **0.551** | 0.804 | **0.932** |

<sub>Macro-averaged over 5 lectures. Generated by `python -m evaluation.benchmark`
→ [`evaluation/results/benchmark.md`](evaluation/results/benchmark.md). Neural
extraction uses a local 7B model (Ollama) — see the model-capacity note below.</sub>

**The finding:** the symbolic engine is the strongest *single* method
(high precision, best concept recall). A small **local** 7B LLM is noisy on its
own — low recall, frequent mistakes on hard OCR-heavy lectures. Yet **fusing the
two still produces the best graph overall** (best edge-F1, recall, and ordering
accuracy): the neural side recovers genuine concepts the rules miss — most
visibly on the recursion lecture, where it lifts edge-F1 from 0.17 → 0.40 — while
the fusion's confidence weighting and direction-conflict resolution stop its
noise from degrading the result (on a lecture where the LLM finds nothing, the
hybrid simply falls back to the symbolic graph). **Model capacity matters:** the
gap widens with a stronger neural model; this table is the honest result with a
7B that runs on commodity hardware (16 GB RAM, 6 GB VRAM).

## Why this is hard (and not just an LLM wrapper)

Three real problems sit underneath the numbers above:

1. **Multi-modal fusion of noisy signals.** The lecturer *speaks* in code-mixed
   Hindi+English and *writes* on a board. Whisper ASR handles the speech;
   Tesseract OCR on handwriting is garbage out of the box. The symbolic
   pipeline (`lecture2graph/symbolic/`) does **confidence-filtered OCR** and
   then **ASR-seeded fuzzy correction** — it builds a domain vocabulary from the
   clean ASR transcript and uses edit-distance (`rapidfuzz`) to repair OCR
   tokens (`"Koot" → "root"`), so it generalizes to new handwriting without a
   per-video correction map.

2. **Prerequisite inference under top-down teaching.** Teachers mention the
   *example* (pre-order) at 0:00 but the *foundational* concept (children) at
   1:45 — so naive temporal ordering is backwards. The DAG builder combines
   domain rules, causal-language anchors ("X means Y", "pehle X phir Y"), and
   temporal edges gated by a **BFS transitive-reachability check** that refuses
   any temporal edge which would contradict an existing dependency path. Cycles
   are broken with Kahn's algorithm by dropping the lowest-confidence edge.

3. **Reconciling two extractors that disagree on *everything*.** The symbolic
   and neural pipelines emit different JSON schemas (`source/target` vs
   `from/to`), different confidence semantics, and different names for the same
   concept (`"breadth first search"` vs `"bfs"`, `"binary tree"` vs
   `binary_tree`). `lecture2graph/graphs.py` is the **entity-resolution layer**:
   a deterministic canonicalizer + alias table that collapses both into one node
   space — so the fusion and the benchmark compare apples to apples.

## Architecture

```
                    YouTube lecture URL
                            │
              ┌─────────────┴─────────────┐
              ▼  ingest → ASR + OCR → normalize (shared M1–M3)
              │  (Whisper · Tesseract · rapidfuzz)
              ▼
      ┌───────────────┐         ┌───────────────┐
      │  SYMBOLIC     │         │  NEURAL        │
      │  regex + domain         │  local LLM     │
      │  rules + DAG  │         │  (Ollama)      │
      └───────┬───────┘         └───────┬────────┘
        graph.symbolic.json       graph.neural.json
              └───────────┬───────────────┘
                          ▼
                 ┌─────────────────┐     canonicalize · noisy-OR confidence
                 │  HYBRID  fuse() │     direction-conflict resolution
                 │                 │     Kahn cycle-breaking
                 └────────┬────────┘
                   graph.hybrid.json
                          │
            ┌─────────────┴──────────────┐
            ▼                            ▼
   ┌──────────────────┐        ┌───────────────────┐
   │  EVALUATION      │        │  GRAPH-RAG TUTOR  │
   │  vs gold DAGs    │        │  "what before X?" │
   │  P/R/F1 · GED    │        │  timestamp-grounded│
   └──────────────────┘        └───────────────────┘
```

## Repo layout

```
lecture2graph/
  graphs.py            shared: canonicalization + schema-agnostic PrereqGraph
  symbolic/            rule-based pipeline (ingest→extract→normalize→concepts→prereqs→visualize)
  neural/              local-LLM (Ollama, OpenAI-compatible) concept + prerequisite extraction
  neural/llm.py        thin stdlib client for any local OpenAI-compatible server
  hybrid/fuse.py       neuro-symbolic fusion → DAG
  tutor/tutor.py       graph-RAG study planner over the DAG
evaluation/
  gold/                domain-authored gold prerequisite DAGs (5 videos)
  metrics.py           edge/node P·R·F1, prereq-order accuracy, graph edit distance
  benchmark.py         symbolic vs neural vs hybrid ablation
  results/             generated benchmark tables
tests/                 offline unittest sanity checks
data/                  cached per-video artifacts (transcripts, segments, graphs)
```

## Quickstart

Everything except processing a *new* video runs **offline on cached artifacts** —
no API key, no GPU, no downloads.

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt        # or: pip install -e .

# 1. reproduce the benchmark (symbolic vs neural vs hybrid)
python -m evaluation.benchmark

# 2. fuse symbolic + neural graphs into hybrid DAGs
python -m lecture2graph.hybrid.fuse

# 3. ask the tutor for a study path
python -m lecture2graph.tutor.tutor XRcC7bAtL3c --before pre_order_traversal
python -m lecture2graph.tutor.tutor azXr6nTaD9M --where recursion

# 4. run the tests
python -m unittest discover -s tests
```

Example tutor output (uses the hybrid graph, grounds each concept to its
timestamp in the source video):

```
To learn 'pre_order_traversal', study these first:
  1. tree            [3s]
  2. node            [12s]
  3. binary_tree     [99s]
  ...
  9. tree_traversal  [3s]
```

### Processing a brand-new lecture (needs ffmpeg, tesseract, a local LLM)

The neural pipeline talks to a local OpenAI-compatible server — no API key, no
cloud. With [Ollama](https://ollama.com):

```bash
ollama pull llama3.1:8b && ollama serve     # or any model; configure via env:
# export LECTURE2GRAPH_LLM_MODEL=llama3.1:8b
# export LECTURE2GRAPH_LLM_BASE_URL=http://localhost:11434/v1

python -m lecture2graph.symbolic.pipeline "https://youtu.be/VIDEO_ID"   # symbolic graph
python -m lecture2graph.neural.pipeline   "https://youtu.be/VIDEO_ID"   # neural graph
python -m lecture2graph.hybrid.fuse VIDEO_ID                            # fuse
```

## Evaluation methodology

- **Gold standard** (`evaluation/gold/*.json`): hand-authored prerequisite DAGs
  over the concepts each lecture actually covers. Directed edge `A → B` means
  *"A must be understood before B."*
- **Matching**: predicted and gold graphs are canonicalized, then directed
  prerequisite edges are compared as exact `(source, target)` pairs. Weak
  temporal-precedence edges are excluded.
- **Metrics**: node & edge precision/recall/F1; **prerequisite-order accuracy**
  (of gold "A before B" pairs whose endpoints we predicted, how many the
  topological sort orders correctly); and a symmetric **graph edit distance**.
- **Fair comparison**: all three methods run on every one of the 5 lectures, so
  the headline table is a like-for-like average (no method gets an easier subset).

## Honest limitations

- The head-to-head benchmark is small (5 lectures) — a real signal, not a large
  one; treat it as a pilot rather than a statistically powered study.
- The neural extractor here is a **7B** model chosen to run on commodity
  hardware. It is weak and high-variance on noisy, OCR-heavy lectures (on two of
  the five it contributes no usable edges). A larger model would make the neural
  and hybrid numbers materially better; the architecture is unchanged.
- Gold graphs are author-annotated (single annotator); they encode defensible
  CS prerequisites but are not multiply-adjudicated.
- Domain rules and concept patterns are CS-specific; generalizing the symbolic
  side to new subjects would need new domain packs (see
  [`docs/symbolic-hardcoding.md`](docs/symbolic-hardcoding.md)).

## Provenance

This began as a research-lab recruitment task (a code-mixed multimodal concept
extractor) and was rebuilt into a measured neuro-symbolic system. The two
extraction pipelines carry that research lineage; the fusion, evaluation
harness, and graph-RAG tutor are the new contributions.
